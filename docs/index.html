10 — scrollProgress()

Track element read progress.

Measure how far through a specific element the user has scrolled — 0 when it enters the viewport, 1 when it leaves. Scroll the article box below.

Scroll snap kit gives you precise control over every aspect of scroll behaviour in your web app. Whether you need smooth navigation, progress tracking, or choreographed animations, the API is designed to be minimal and composable.

The scrollProgress utility fires a callback with a normalised value from 0 to 1 as the user scrolls through the target element. This makes it trivially easy to build reading progress bars, reveal animations, or sticky sidebars that respond to content position.

Unlike scroll percentage of the whole page, scrollProgress is scoped to a single element. This means you can independently track multiple articles, carousels, or code blocks on the same page without any interference.

Under the hood it uses getBoundingClientRect on every scroll event, with a lightweight requestAnimationFrame throttle to keep performance smooth even on long pages.

Combined with the easeScroll and scrollSequence utilities, scrollProgress unlocks scroll-driven storytelling patterns that previously required heavyweight libraries.
